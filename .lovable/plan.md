

## Auto-generate "Processo Interno" Code on New Project

### Overview

When creating a new project, auto-generate the `processo_uvv` field using format `SIGLA-ANO-SEQUENCIAL` (e.g. `UVV-2026-0001`). The sigla comes from the user's `fomento_organizations` record (already has a `sigla` column). The field is editable but visually indicates it was auto-generated.

### Architecture Note

The `fomento_organizations` table already has a `sigla` column. No need to add `sigla` to `profiles` — we fetch it via the user's `fomento_org_id`. For the admin screen, we add an inline edit for the organization's `sigla`.

### Changes

**1. Database: Create a SQL function `generate_fomento_processo`**

A `SECURITY DEFINER` function that:
- Accepts `_org_id uuid`
- Looks up `sigla` from `fomento_organizations` (fallback: `'ORG'`)
- Counts `fomento_projects` for that org in the current year + 1
- Returns `SIGLA-YEAR-NNNN`

This ensures atomic, consistent sequence generation.

**2. `src/contexts/FomentoAuthContext.tsx`** — No changes needed (already exposes `fomentoOrgId`).

**3. `src/components/fomento/FomentoProjectForm.tsx`**

- On mount (new project only, not editing), call `supabase.rpc('generate_fomento_processo', { _org_id })` to get the auto-generated code
- Pre-fill `processo_uvv` with the result
- Track `autoGenerated` boolean state
- Render the field with a light blue background (`bg-blue-50`) and a small badge "Gerado automaticamente" when auto-generated
- Field remains editable; editing clears the auto-generated indicator

**4. `src/components/fomento/FomentoAdmin.tsx`**

- Fetch the current organization's `sigla` from `fomento_organizations`
- Add a card section "Configuração da Organização" with an input for `sigla`
- Save via `supabase.from('fomento_organizations').update({ sigla }).eq('id', orgId)`
